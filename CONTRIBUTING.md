# Contributing to Chat402 Community

Thank you for your interest in contributing to Chat402! We welcome contributions from developers of all skill levels.

## How to Contribute

### 1. Code Examples

Share examples in your favorite programming language:

- Create a new file in `/examples/{language}/`
- Follow the existing structure
- Include clear comments
- Add a README explaining the example

**Example structure:**
```
examples/
  python/
    basic_example.py
    streaming_example.py
    README.md
  javascript/
    basic_example.js
    README.md
```

### 2. SDKs & Libraries

Building a Chat402 SDK? We'd love to feature it!

- Add it to `/sdks/{language}/`
- Include documentation
- Provide installation instructions
- Add tests if possible

### 3. Documentation

Help improve our docs:

- Fix typos or unclear explanations
- Add missing examples
- Translate documentation
- Create tutorials

### 4. Bug Reports

Found a bug? Please open an issue with:

- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Code snippet (if applicable)

## Guidelines

### Code Style

- Use clear, descriptive variable names
- Add comments for complex logic
- Follow language-specific conventions
- Keep examples simple and focused

### Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-example`)
3. Make your changes
4. Test your code
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

### PR Template

```markdown
## Description
Brief description of what this PR adds

## Type of Contribution
- [ ] Code example
- [ ] SDK/Library
- [ ] Documentation
- [ ] Bug fix

## Testing
How did you test this?

## Screenshots (if applicable)
Add screenshots for UI changes
```
